security & offensive tooling

Most of my security knowledge comes from hands-on practice rather than formal courses — working through OWASP Top 10 categories, building and breaking tools on ESP32 hardware, and treating a no-root, non-jailbroken Android phone via Termux as a deliberate constraint rather than a limitation.

Comfortable across network penetration testing, firewall evasion, pivoting, AV evasion, ethical malware development, firmware reverse engineering, and exploit development — particularly API and injection vulnerabilities with high-impact potential. Binary/low-level exploitation (protection bypasses, etc.) is the area I'm still actively building depth in.

When a tool I need doesn't exist, I build it — NRSuite, DuckESP, ESP-Bridge, and BridgeProtocol all started that way. Also contributed to Bruce, a predatory ESP32 firmware project.

Additional offensive tooling developed for private research and testing — not publicly distributed.

Termux is basically a second daily driver at this point — I've pushed it to the point of running full workflows without ever touching a laptop.

Two responsibly-disclosed findings on a nationwide vendor's systems, kept intentionally vague out of respect for the vendor:

  • Unauthenticated admin account creation vulnerability (2021)
  • Critical remote code execution via command injection, leading to full system compromise with root privileges (2022) — reported privately, no public details disclosed
